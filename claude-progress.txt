## 2026-02-20 - 实现网页抓取工具（解决 agent_max 与工具冲突问题）

### 完成内容:
- 创建 `WebExtractorTool` 工具类 (`src/schedule_agent/core/tools/web_extractor_tool.py`)
  - 作为独立的工具，通过工具调用方式实现网页抓取
  - 工具内部使用独立的 LLM 调用（启用网页抓取，但不传其他工具）
  - 解决了 agent_max 模式与自定义工具冲突的问题

- 修改 `LLMClient`
  - 移除全局的 `enable_web_extractor` 逻辑（避免与工具冲突）
  - 保留 `_chat_with_tools_stream` 方法供工具内部使用
  - 全局调用不再使用 agent_max 策略（避免冲突）

- 更新工具注册 (`main.py`, `__init__.py`)
  - 在 `get_default_tools` 中添加 `WebExtractorTool`
  - 仅当 provider=qwen 时添加网页抓取工具

- 更新系统提示词 (`agent.py`)
  - 添加网页访问能力说明
  - 告知 Agent 可以使用 `extract_web_content` 工具

- 编写测试用例
  - `tests/test_web_extractor_tool.py`: 完整的工具测试
  - 更新 `tests/test_llm.py`: 移除全局网页抓取测试
  - 所有测试通过（18 passed, 1 skipped）

### 技术实现:
- **解决方案**: 将网页抓取功能封装为独立工具，避免 agent_max 模式与自定义工具冲突
- **工作原理**:
  1. 用户请求网页内容时，Agent 调用 `extract_web_content` 工具
  2. 工具内部创建独立的 LLM 调用（enable_search + enable_thinking + search_strategy: agent_max）
  3. 工具调用时不传其他工具，避免冲突
  4. 使用流式模式（网页抓取要求）
  5. 返回提取的网页内容

- 所有现有测试通过，未破坏现有功能

### 备注:
- 网页抓取工具仅在 provider=qwen 时可用
- 工具会自动处理流式调用和配置
- 解决了 "Agent mode does not support tools" 错误
- 现在可以同时使用日程管理工具和网页抓取功能

   - token 粗略估算（中英文混合）
   - 滑动窗口阈值检测（可配置 max_working_tokens + threshold 比例）
   - LLM 驱动窗口总结：旧消息折叠为摘要块，保留最近 N 轮原始消息
   - 会话结束总结：输出结构化 JSON（summary / decisions / open_questions / referenced_files / tags）

3. **短期记忆 (ShortTermMemory)**:
   - JSONL 持久化 FIFO 队列（容量 K 可配）
   - 关键词搜索 + 格式化注入 system prompt
   - 出队时返回被淘汰条目供长期记忆提炼

4. **长期记忆 (LongTermMemory)**:
   - LLM 提炼出队的短期记忆，生成 MemoryEntry 条目
   - JSONL 持久化 + Markdown 文件双写
   - 自动追加 MEMORY.md（高确信条目标记"待人工确认"）
   - 初始化 MEMORY.md 模板（用户偏好 / 约束 / 决策 / 教训）
   - QMD collection 写入支持（qmd_enabled 开关）

5. **内容记忆 (ContentMemory)**:
   - markitdown 格式转换（PDF / DOCX → Markdown）
   - 按分类目录存储（docs / meeting / diary / lessons / notes / code）
   - 关键词搜索 + QMD 语义检索

6. **记忆检索策略 (RecallPolicy)**:
   - 强制 / 模式匹配两种触发模式
   - 多路检索：短期 + 长期 + 内容 + MEMORY.md
   - RecallResult 聚合，格式化注入 system prompt

7. **Agent 集成**:
   - 在 process_input 前执行记忆检索，将结果注入系统提示
   - 每轮 LLM 调用前检查工作记忆窗口，必要时触发总结
   - 会话结束时 finalize_session：总结 → 短期入队 → 出队提炼
