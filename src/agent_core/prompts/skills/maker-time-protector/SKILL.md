# Maker Time Protector

## 🎯 核心理念

本技能基于 **Cal Newport《Deep Work》** 和 **Time Blocking** 方法，帮助用户：

1. **区分 Maker Time 和 Manager Time**
   - **Maker Time**：深度工作、写代码、做项目、学习的大块连续时间（至少 2 小时）
   - **Manager Time**：课程、会议、琐事、回复消息等碎片时间

2. **主动设计理想的一周**，而不是被动响应日程

3. **保护专注时间**，对干扰说"不"

---

## ⏰ 用户画像与默认设置

**用户习惯**：
- 晚睡晚起：通常凌晨才睡，中午左右起床
- 上海交通大学计算机系学生
- 需要大块时间写代码、做项目

**默认专注时间块建议**（可根据实际情况调整）：
- **下午块**：14:00 - 18:00（4 小时）
- **晚上块**：20:00 - 24:00（4 小时）
- **深夜块**（可选）：00:00 - 02:00（2 小时，适合深夜高效者）

## 🚀 核心工作流

### Workflow 1: 设计理想周（Ideal Week Design）

**触发**：用户说"设计我的一周"、"理想周模板"、"规划这周的时间块"

**步骤**：

1. **收集固定承诺**
   - 询问用户这周有哪些不可变动的时间（课程、例会、固定约定）
   - 调用 `get_events` 查询已有日程

2. **识别最佳思考时间**
   - 询问用户什么时候最清醒、最适合深度工作
   - 默认推荐：14:00-18:00, 20:00-24:00（考虑晚睡晚起习惯）

3. **生成理想周模板**
   - 在空闲时间段中建议创建 Maker Time 块
   - 使用 `get_free_slots` 查找可用时间
   - 使用 `add_event` 创建专注时间块事件（带标签 "Maker Time"）

4. **设置保护机制**
   - 为 Maker Time 块设置高优先级
   - 添加描述："保护这段时间用于深度工作，非紧急不打扰"
   - 提供拒绝话术模板

**输出示例**：

```
📅 你的理想周模板（2026-02-23 ~ 2026-03-01）

固定承诺：
- 周一 09:00-10:00 课程
- 周三 14:00-16:00 组会

建议的 Maker Time 块：
- 周一 14:00-18:00 【深度工作】写项目代码
- 周二 14:00-18:00 【深度工作】算法学习
- 周三 19:00-23:00 【深度工作】项目收尾
- 周四 14:00-18:00 【深度工作】课程作业
- 周五 20:00-24:00 【深度工作】自由探索

已为你创建 5 个 Maker Time 时间块，总计 20 小时。
```

### Workflow 2: 每日时间块规划（Daily Time Block Planning）

**触发**：用户说"规划今天"、"今天的时间块怎么安排"、"今天有哪些 Maker Time"

**步骤**：

1. 调用 `get_events` 查询今日日程
2. 调用 `get_free_slots` 查找今日空闲时间
3. 识别可用于 Maker Time 的连续时间段（≥2 小时）
4. 询问用户今天想完成什么深度任务
5. 使用 `add_event` 创建今日 Maker Time 块

**输出示例**：

```
📅 今日时间块规划（2026-02-27）

已有日程：
- 10:20-10:28 致远学院资格审查

空闲时间段：
- 12:00-14:00（2 小时）
- 14:30-18:00（3.5 小时）⭐ 推荐 Maker Time
- 19:00-23:00（4 小时）⭐ 推荐 Maker Time

建议安排：
1. 【Maker Time】14:30-18:00 - 完成 XXX 项目核心功能
2. 【Maker Time】20:00-22:00 - 复习算法

要帮你创建这些时间块吗？
```

### Workflow 3: Maker Time 分析（Maker Time Analysis）

**触发**：用户说"这周 Maker Time 够吗"、"分析我的时间块"、"我有多少专注时间"

**步骤**：

1. 调用 `get_events` 查询本周日程
2. 筛选带 "Maker Time" 标签的事件
3. 计算总 Maker Time 时长
4. 与推荐值对比（建议每周至少 15-20 小时深度工作）
5. 给出改进建议

**输出示例**：

```
📊 本周 Maker Time 分析（2026-02-23 ~ 2026-03-01）

已安排的 Maker Time：
- 周一 14:00-18:00（4 小时）
- 周三 19:00-23:00（4 小时）
- 周五 14:00-17:00（3 小时）

总计：11 小时

⚠️ 低于推荐值（15-20 小时/周）

建议：
1. 周二下午还有 3 小时空闲，可以安排一个 Maker Time 块
2. 周六上午 10:00-12:00 也可利用

要帮你补充这些时间块吗？
```

### Workflow 4: 时间块保护（Focus Block Protection）

**触发**：用户说"有人约我这个时间"、"这个时间能不能开会"、"如何保护专注时间"

**提供拒绝话术模板**：

```
🛡️ 保护 Maker Time 的话术模板：

【温和但坚定】
"谢谢邀请！不过这个时间段我已经安排了深度工作，是专门留给 XXX 项目的。
可以改到 [替代时间] 吗？"

【直接版】
"这个时间是我的专注时间块，不安排会议。其他时间可以吗？"

【提前声明】（适合 recurring Maker Time）
"我每周二、四下午是固定深度工作时间，不安排会议。其他时间欢迎预约！"
```

---

### Workflow 5: 周度复盘（Weekly Review）

**触发**：用户说"时间块复盘"、"这周专注时间执行得怎么样"、"每周复盘"

**步骤**：

1. 调用 `get_events` 查询本周已完成的 Maker Time 事件
2. 对比计划 vs 实际执行
3. 识别被打断/取消的时间块
4. 生成改进建议

**输出示例**：

```
🔄 本周时间块复盘（2026-02-17 ~ 2026-02-23）

计划 Maker Time：20 小时
实际完成：14 小时
完成率：70%

被打断的时间块：
- 周三 19:00-23:00 → 只完成了 2 小时（组会延长）
- 周五 14:00-18:00 → 取消（朋友约饭）

改进建议：
1. 周三晚上的时间块容易被组会影响，建议改到周四
2. 周五下午是社交高发期，可以主动安排社交而非被动响应

🎯 下周目标：保持 15 小时 Maker Time，减少被打断次数
```

## 📋 最佳实践

### 1. Maker Time 设置原则

- **最少 2 小时**：深度工作需要至少 90 分钟进入状态
- **固定时段**：每周 recurring 的时间块更容易坚持
- **高优先级**：设为 `priority: high` 或 `urgent`
- **清晰描述**：写明具体任务（如"完成 XXX 项目核心功能"而非"工作"）

### 2. 保护策略

- **提前声明**：在团队/朋友间公开你的 Maker Time 安排
- **关闭通知**：Maker Time 期间关闭消息通知
- **物理隔离**：去图书馆、实验室等专注环境
- **替代方案**：拒绝时提供替代时间，减少关系成本

### 3. 灵活调整

- 如果某天状态不好，可以缩短 Maker Time（如 4 小时→2 小时）
- 如果某天状态好，可以延长或增加深夜块
- 每周复盘时根据实际情况调整模板

## ⚠️ 注意事项

1. **不要过度安排**：每周 Maker Time 建议 15-20 小时，留有余地
2. **允许灵活调整**：状态不好时可以缩短，不要有罪恶感
3. **复盘比计划重要**：每周复盘执行情况，持续优化模板
4. **社交也重要**：不要把所有时间都变成 Maker Time，保留社交和休息时间
